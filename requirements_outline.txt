requirements outline

functional requirements
1) add edit search and delete player accounts
2) add edit search and delete card records
3) create and manage player decks with card assignments
4) create and manage tournaments and entries
5) produce admin reports and summary statistics
6) keep audit friendly timestamps for key records

non functional requirements
1) usability
- simple menu based navigation with clear prompts and feedback
2) performance
- common search operations should return quickly on school scale datasets
3) reliability
- transactions should preserve consistency during write operations
4) security
- staff access should be restricted with authentication controls in the interface layer
5) maintainability
- modular python functions with consistent naming and reusable database helpers
6) data integrity
- enforce foreign keys and validation rules to reduce duplicates null critical fields and invalid ranges
