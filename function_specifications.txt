function specifications

connect_db(path) -> connection
- purpose: open sqlite connection and enable foreign keys
- params: path (string)
- returns: sqlite connection
- processing: connect, set pragmas, return

create_player(conn, name, email) -> player_id or None
- purpose: insert a new player with validation
- params: conn, name (string), email (string optional)
- returns: new player_id or None on error
- processing: validate name not empty, parameterised insert, commit

search_player(conn, term) -> list of rows
- purpose: find players by name or email
- params: conn, term (string)
- returns: list of matching player dicts
- processing: parameterised like query lower(name) or email

create_card(conn, name, set_id, rarity, card_type, mana_cost) -> card_id
- purpose: add card to catalogue
- params: listed
- returns: card_id
- processing: validate set exists, insert, commit

create_deck(conn, player_id, name) -> deck_id
- purpose: create user deck record
- processing: validate player exists, insert, commit

add_card_to_deck(conn, deck_id, card_id, quantity)
- purpose: add or update quantity of a card in a deck
- processing: use transaction, upsert-like logic (select then insert or update), commit

create_tournament(conn, name, start_date, format) -> tournament_id
register_player_tournament(conn, tournament_id, player_id) -> entry_id
generate_report_tournament_entries(conn, tournament_id) -> rows

notes
- all db writes use parameterised queries
- exceptions propagated to caller and handled at UI layer
